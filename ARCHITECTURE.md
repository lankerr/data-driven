## 1. CNN 骨干网络的选择

### 我们现在用的：ResNet34（默认） / ResNet50（可选）

**❌ 为什么不用 ResNet152？**

这是一个非常好的问题。直觉上觉得"越深越好"，但在我们这个任务里恰恰相反：

```
┌─────────────────────────────────────────────────────────────────┐
│                    ResNet 家族对比                                │
├──────────┬──────────┬───────────┬──────────────────────────────┤
│  型号    │ 参数量   │ 层数      │ 特点                          │
├──────────┼──────────┼───────────┼──────────────────────────────┤
│ ResNet18 │  11.7M   │  18层     │ 太浅，特征不够丰富              │
│ ResNet34 │  21.8M   │  34层     │ ✅ 性价比最高，轻快且够用        │
│ ResNet50 │  25.6M   │  50层     │ ✅ 可选，Bottleneck结构更强     │
│ ResNet101│  44.5M   │ 101层     │ 太重，显存吃不消               │
│ ResNet152│  60.2M   │ 152层     │ ❌ 巨大，训练慢，收益递减       │
└──────────┴──────────┴───────────┴──────────────────────────────┘
```

**核心原因有三个：**

**(a) 我们不是在做 ImageNet 分类！**

ResNet152 在 ImageNet 上比 ResNet34 强 ~2%，因为它需要区分1000种物体的细微差异（比如区分100种狗）。但我们的任务是**像素级预测**——预测每个像素未来的雷达回波值。这需要的是多尺度空间特征，而不是极深的语义理解。

**(b) 编码器太深 = 解码器恢复不了细节**

```
ResNet34 的特征金字塔 (简洁高效):
    输入 128×128
      ↓
    ┌──────┐
    │ 32×32 │ ← layer1: 64通道   (浅层特征: 边缘、纹理)
    │ 16×16 │ ← layer2: 128通道  (中层特征: 回波轮廓)
    │  8×8  │ ← layer3: 256通道  (深层特征: 回波运动模式)
    │  4×4  │ ← layer4: 512通道  (最深: 全局信息)
    └──────┘

ResNet152 的问题:
    输入 128×128
      ↓
    ┌──────┐
    │ 32×32 │ ← 256通道 (Bottleneck)  ← 通道数暴增!
    │ 16×16 │ ← 512通道
    │  8×8  │ ← 1024通道
    │  4×4  │ ← 2048通道              ← 4×内存开销!
    └──────┘
    解码器要从 2048 通道恢复到像素级预测，难度大增
```

**(c) 我们的输入只有 128×128**

128×128 的图像经过4次下采样就变成 4×4，空间信息已经很少了。152层网络在 4×4 的特征图上堆了大量冗余计算。

### 🔧 如果想换更强的 CNN？

在配置文件里改一行即可：

```yaml
# config.yaml
model:
  resnet_backbone: "resnet50"    # 从 resnet34 换成 resnet50
```

ResNet50 使用 **Bottleneck 结构**（1×1→3×3→1×1 三层卷积），在参数量增加不多的情况下提升了表达能力，是一个合理的升级选择。

> 💡 **结论**：对于 128×128 的气象回波，**ResNet34 是最佳性价比选择**，ResNet50 作为进阶。ResNet152 反而因为太深太重导致训练困难、过拟合风险高、显存爆炸。

## 2. ViT 骨干网络的选择

### 我们现在用的：Swin Transformer（Swin-T）

**❌ 为什么不用原版 ViT (Vision Transformer)？**

原版 ViT 有一个致命问题：**它只输出一个分辨率的特征，不适配 UNet！**

```
┌──────────────────────────────────────────────────────────────┐
│               原版 ViT vs Swin Transformer                    │
├──────────────────────┬───────────────────────────────────────┤
│    原版 ViT          │     Swin Transformer                   │
│                      │                                        │
│  输入: 128×128       │  输入: 128×128                         │
│     ↓ 切patch        │     ↓ 切patch                          │
│  [16×16] = 64个patch │  Stage1: 32×32 (96ch)  ← f1 跳跃连接   │
│     ↓                │     ↓                                  │
│  12层 Transformer    │  Stage2: 16×16 (192ch) ← f2 跳跃连接   │
│     ↓                │     ↓                                  │
│  输出: 64个token     │  Stage3:  8×8  (384ch) ← f3 跳跃连接   │
│  (只有1种分辨率!)     │     ↓                                  │
│                      │  Stage4:  4×4  (768ch) ← f4 瓶颈层     │
│  ❌ 没法做 UNet!     │                                        │
│                      │  ✅ 天然4级金字塔，完美适配 UNet!        │
└──────────────────────┴───────────────────────────────────────┘
```

**Swin Transformer 的三大优势：**

1. **层次化特征金字塔**：像 ResNet 一样逐级降分辨率，天然适配 UNet 的跳跃连接
2. **窗口注意力**：计算量从 O(N²) 降到 O(N)，128×128 图像也跑得动
3. **移位窗口**：窗口间有信息交换，保留全局建模能力

```
Swin Transformer 的窗口注意力机制:

┌───┬───┬───┬───┐       ┌───┬───┬───┬───┐
│ A │ A │ B │ B │       │ · │ D │ D │ · │
├───┼───┼───┼───┤  移位  ├───┼───┼───┼───┤
│ A │ A │ B │ B │  ──→  │ · │ D │ D │ · │
├───┼───┼───┼───┤       ├───┼───┼───┼───┤
│ C │ C │ D │ D │       │ · │ B │ B │ · │
├───┼───┼───┼───┤       ├───┼───┼───┼───┤
│ C │ C │ D │ D │       │ · │ B │ B │ · │
└───┴───┴───┴───┘       └───┴───┴───┴───┘
 层1: 窗口内注意力         层2: 移位后注意力
 (局部特征)              (跨窗口交互=全局信息)
```

### Swin-T 和 Swin-S 的区别

```
┌─────────┬──────────┬───────┬─────────────────────────┐
│  型号   │ 参数量   │ 通道  │ 各阶段 Transformer 块数   │
├─────────┼──────────┼───────┼─────────────────────────┤
│ Swin-T  │  28.3M   │ 96    │ [2, 2, 6, 2]            │
│ Swin-S  │  49.6M   │ 96    │ [2, 2, 18, 2]           │
│ Swin-B  │  87.8M   │ 128   │ [2, 2, 18, 2]           │
└─────────┴──────────┴───────┴─────────────────────────┘
```

> 💡 **结论**：Swin-T 是"ViT世界里的ResNet34"——轻量、高效、刚好够用。Swin-S 可以作为进阶选择。

## 3. 整体架构：UNet 编码器-解码器

**UNet 才是我们所有实验的核心骨架**。ResNet 和 Swin 只是被"拆开"后塞进 UNet 的编码器部分。

```
完整的网络数据流 (以 ResNet-UNet 为例):

     输入: 13帧雷达回波                                    输出: 13帧预测
     (B, 13, 128, 128)                                    (B, 13, 128, 128)
           │                                                    ▲
           ▼                                                    │
    ┌─────────────┐                                    ┌─────────────┐
    │ 1×1 Conv    │  13通道 → 3通道                      │  1×1 Conv   │
    │ (适配器)     │  (匹配ImageNet预训练)                 │  (输出头)    │
    └──────┬──────┘                                    └──────┬──────┘
           │                                                    ▲
           ▼                                                    │
    ══════════════════════ UNet 主体 ══════════════════════════
    │                                                          │
    │  编码器 (下采样)              解码器 (上采样)              │
    │  ┌────────────┐              ┌────────────┐              │
    │  │ 128→32×32  │──跳跃连接──→│ 32×32→128  │              │
    │  │ ResNet L1  │    f1        │ UpBlock 3  │              │
    │  │  64通道    │              │            │              │
    │  └─────┬──────┘              └─────▲──────┘              │
    │        │                           │                      │
    │  ┌─────▼──────┐              ┌─────┴──────┐              │
    │  │ 32→16×16   │──跳跃连接──→│ 16→32×32   │              │
    │  │ ResNet L2  │    f2        │ UpBlock 2  │              │
    │  │ 128通道    │              │            │              │
    │  └─────┬──────┘              └─────▲──────┘              │
    │        │                           │                      │
    │  ┌─────▼──────┐              ┌─────┴──────┐              │
    │  │ 16→8×8     │──跳跃连接──→│ 8→16×16    │              │
    │  │ ResNet L3  │    f3        │ UpBlock 1  │              │
    │  │ 256通道    │              │            │              │
    │  └─────┬──────┘              └─────▲──────┘              │
    │        │                           │                      │
    │        ▼         瓶颈层            │                      │
    │  ┌────────────────────────────────┐                      │
    │  │  ResNet L4: 4×4, 512通道       │                      │
    │  │  (最深层特征: 全局抽象信息)      │                      │
    │  └────────────────────────────────┘                      │
    │                                                          │
    ══════════════════════════════════════════════════════════
```

### 跳跃连接的关键作用

想象一下你在画一幅画：
- **编码器**像是在不断缩小画布总结概要："有一团云在东北方向移动"
- **瓶颈层**是最高层的总结："全局的大气运动方向"
- **解码器**要把这些总结还原成精细的画面
- **跳跃连接**相当于把每一步缩小前的"草稿"直接传过去，这样解码器不用凭空想象细节

```
没有跳跃连接:                     有跳跃连接:
  编码器 → 瓶颈 → 解码器            编码器 ──→ 解码器
  "有云"  "向东"  "模糊的云"        128×128细节 + "向东" = 精确的云
                                   
  结果: 模糊!                      结果: 边缘清晰!
```

## 4. 数据流关键技巧：Time-as-Channel

**问题**：预训练的 ResNet/Swin 接受 3 通道 RGB 图像输入，但我们有 **13 帧**时序数据。

**解决方案**：把时间维度当成通道维度！

```
原始输入:
┌─────┐ ┌─────┐ ┌─────┐     ┌─────┐
│ t=1 │ │ t=2 │ │ t=3 │ ... │t=13 │    13帧, 每帧 128×128
└─────┘ └─────┘ └─────┘     └─────┘
    ↓ 堆叠成 (B, 13, 128, 128)

         ┌────────────────┐
         │  1×1 卷积适配器  │    13通道 → 3通道
         │  ChannelAdapter │    可学习的线性组合
         └────────┬───────┘
                  ↓
         (B, 3, 128, 128)     现在可以喂给预训练模型了!

类比理解:
  1×1 卷积做的事情 ≈ "把13帧压缩成3张最重要的摘要图"
  channel_1 ≈ 学到了"回波位置信息" (类似R通道)
  channel_2 ≈ 学到了"回波运动方向" (类似G通道)  
  channel_3 ≈ 学到了"回波强度变化" (类似B通道)
```

## 5. 五组实验详解

### 实验 1：纯 ResNet-UNet

```
📝 一句话: 用 CNN 的眼睛看世界，擅长局部细节

输入(13帧) → [1×1 Conv] → ResNet34编码器 → UNet解码器 → 输出(13帧)

           ResNet 的视野:
           ┌─────────────┐
           │ ██ ← 3×3卷积│   每一层只看周围的小区域
           │ ██   窗口    │   层层堆叠才能看到远处
           └─────────────┘
           
✅ 优点: 回波边缘锐利, 局部纹理细腻
❌ 缺点: 对远距离的回波平流预测不够准确
         (一个城市的暴风不知道另一个城市的云在干什么)
         
参数量: 22,776,007 (~22.8M)
```

### 实验 2：纯 ViT-UNet (Swin Transformer)

```
📝 一句话: 用 Transformer 的眼睛看世界，擅长全局关系

输入(13帧) → [1×1 Conv] → Swin-T编码器 → UNet解码器 → 输出(13帧)

           Swin 的视野:
           ┌─────────────────────┐
           │  ↔  ↔  ↔  ↔  ↔  ↔ │   注意力机制让每个位置
           │  ↕  ↕  ↕  ↕  ↕  ↕ │   都能看到其他位置
           │  全局信息交换       │
           └─────────────────────┘
           
✅ 优点: 回波整体移动方向和位置预测准确
❌ 缺点: 受 patch 化影响，局部细节可能有马赛克感
         (知道云要去哪，但画不清楚云的边缘)
         
参数量: 30,871,849 (~30.9M)
```

### 实验 3：ResNet 打底 + ViT 修残差

```
📝 一句话: CNN 先画草稿，Transformer 做全局调整

                    ┌───────────────────┐
                    │ ResNet-UNet       │
   输入(13帧) ─────→│ (画草稿)          │──→ Y_base (基础预测)
        │           └───────────────────┘           │
        │                                           │
        │  ┌────────────────────────────────────────┘
        │  │
        ▼  ▼
  ┌──────────────┐
  │ 拼接: [输入,  │    26通道输入
  │     Y_base]  │
  └──────┬───────┘
         ▼
  ┌───────────────────┐
  │ Swin-UNet         │
  │ (全局修正残差)     │──→ ΔY (残差)
  └───────────────────┘
  
  最终输出 = Y_base + ΔY

💡 直觉: 
  CNN: "暴风大概在这里，形状大概是这样" (局部轮廓)
  ViT: "不对，根据全局气流，应该往东偏移20公里" (全局修正)
  
参数量: 53,647,895 (~53.6M)
```

### 实验 4：ViT 打底 + ResNet 修残差 ⭐ 最推荐

```
📝 一句话: Transformer 先定大方向，CNN 补局部细节

                    ┌───────────────────┐
                    │ Swin-UNet         │
   输入(13帧) ─────→│ (全局趋势预测)    │──→ Y_base (基础预测)
        │           └───────────────────┘           │
        │                                           │
        │  ┌────────────────────────────────────────┘
        │  │
        ▼  ▼
  ┌──────────────┐
  │ 拼接: [输入,  │    26通道输入
  │     Y_base]  │
  └──────┬───────┘
         ▼
  ┌───────────────────┐
  │ ResNet-UNet       │
  │ (局部细节修补)     │──→ ΔY (残差)
  └───────────────────┘
  
  最终输出 = Y_base + ΔY

💡 直觉 (对应真实气象物理):
  第一步 - ViT全局预测:
    "根据大气环流，整个云系会向东北方向移动60公里"
    "整体降水强度会增强"
    → 类比: 天气预报中的大尺度系统预报
    
  第二步 - ResNet局部修补:
    "在 Y_base 预测的基础上..."
    "这里有一个新生对流单体，需要补上一个强回波中心"
    "那条飑线的前沿需要画得更锐利"
    → 类比: 临近预警中的局地强对流修正
    
  Y_final = 大尺度预测 + 局部修正

  ⭐ 这个方案最符合气象物理:
     先有大气环流驱动 (全局) → 再有局部对流爆发 (细节)
  
参数量: 53,647,895 (~53.6M)
```

### 实验 5：混合融合 (Hybrid UNet)

```
📝 一句话: 两个大脑同时工作，门控机制决定谁说的对

                   ┌───────────────┐
                   │ ResNet编码器   │
   输入(13帧) ────→│ (局部特征)     │──→ [f1_r, f2_r, f3_r, f4_r]
        │          └───────────────┘              │
        │                                         │ 逐级门控融合
        │          ┌───────────────┐              │
        └────────→│ Swin编码器    │──→ [f1_s, f2_s, f3_s, f4_s]
                   │ (全局特征)     │              │
                   └───────────────┘              │
                                                  ▼
                               ┌─────────────────────────────┐
                               │     门控融合 (每个尺度)       │
                               │                             │
                               │  gate = σ(W·[fr, fs])       │
                               │  f_fused = g·fr + (1-g)·fs  │
                               │                             │
                               │  g→1: 相信CNN (纹理重要)     │
                               │  g→0: 相信Transformer (位置) │
                               │  g=0.5: 各取所长              │
                               └──────────┬──────────────────┘
                                          ▼
                               ┌─────────────────┐
                               │  UNet解码器      │──→ 输出(13帧)
                               └─────────────────┘

💡 直觉:
  在回波边缘区域: gate → 1 (CNN更擅长画边缘)
  在大面积平稳区域: gate → 0 (Transformer更擅长把握全局)
  网络自己学习什么时候该听谁的！
  
  形象比喻: 
    ResNet 是"显微镜" 🔬 —— 看得清细节
    Swin 是"望远镜" 🔭 —— 看得远全局
    门控机制是"项目经理" 👨‍💼 —— 决定这个区域该听谁的意见

参数量: 55,033,203 (~55.0M)
```

## 6. 损失函数

### 为什么不能用普通 MSE？

```
真实的雷达回波分布:

  像素数量
  ▲
  ███████████
  ██████████│
  █████████ │
  ████████  │
  ███████   │
  ██████    │
  █████     │
  ████      │
  ███       │     ← 我们最关心的暴雨区域
  ██        │        只占 <1% 的像素
  █    █    │        但对预警至关重要！
  ─────────────→ 回波强度
  无雨  弱  中  强  暴雨

  如果用 MSE: 模型会学到"预测全无雨，损失就很低"
  因为 99% 的像素确实没有雨！
```

**我们的方案：B-MSE (Balanced MSE) + SSIM**

```
B-MSE: 按降水强度给权重

  VIL值     │ 对应天气   │  MSE权重
  ──────────┼───────────┼─────────
  < 16      │ 无/微量    │  ×1    (不太关心)
  16 ~ 74   │ 小雨      │  ×2
  74 ~ 133  │ 中雨      │  ×5
  133 ~ 160 │ 大雨      │  ×10
  > 181     │ 暴雨/冰雹  │  ×30   (最关心！)

  效果: 暴雨区域的预测误差被放大30倍，
        迫使模型认真预测这些关键区域！

SSIM Loss: 保持回波"长得像"

  仅有像素误差:              加上 SSIM:
  ┌─────────────┐           ┌─────────────┐
  │  ○  (真实)   │           │  ○  (真实)   │
  │  ◎  (预测)   │           │  ○  (预测)   │
  │  位置对了    │           │  形状轮廓    │
  │  但边缘模糊  │           │  都保持一致  │
  └─────────────┘           └─────────────┘

总损失 = 0.7 × B-MSE + 0.3 × SSIM_Loss
```

## 💡 总结回答你的问题

| 问题 | 回答 |
|------|------|
| CNN用的什么？ | **ResNet34**（默认），可选 ResNet50。不用 ResNet152 因为太重且收益小 |
| ViT用的什么？ | **Swin Transformer Tiny**，不用原版 ViT 因为它没有多尺度特征金字塔 |
| 整体架构是什么？ | **UNet**（编码器-解码器 + 跳跃连接），ResNet/Swin 只是被塞进 UNet 编码器的"大脑" |
| 时序怎么处理？ | **Time-as-Channel**：13帧 → 1×1卷积 → 3通道 → 匹配预训练权重 |
| 哪个实验最值得期待？ | **实验4**（ViT→ResNet残差）：先预测全局趋势，再修补局部对流，最符合气象物理 |
